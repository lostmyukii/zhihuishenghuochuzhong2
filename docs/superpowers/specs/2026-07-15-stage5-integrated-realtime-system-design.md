# 阶段5：一体化实时系统设计

## 1. 已确认目标

本阶段不再以“逐个端口人工试鸣/试亮”作为主线，而是一次形成可运行的「N16R8 无摄像头家庭情境侦探屋」系统：

- 真板持续采集八类传感器，每行输出一个完整 JSON。
- 板端运行情境评分、安全优先级和五类执行器联动。
- Chrome/Edge 网页通过 Web Serial 一键连接开发板；Python 网关作为备用。
- Dashboard 实时显示原始数值、数据有效性、证据链、安全原因、计划动作、真实应用状态和同 ID `ack`。
- 保留设备注册、房间热区、六种情境、网页语音、调试台、当前状态和数据日志，但不复制相邻项目的名称、模式或部署标识。

## 2. 固定硬件与供电

GPIO 保持现有合同：传感器 `1/4/14/5/10/2/8/45`，蜂鸣器 `13`，风扇 PWM `11`，舵机 `9`，继电器 `12`，RGB 灯环 `46`。RFID 禁用。

风扇采用已确认的供电方案：

```text
外部电池 VM -> 风扇 VM
外部电池 GND -> 风扇 GND -> 主板 GND（共地）
主板 GPIO11 -> 风扇 PWM
主板 VM 不接风扇 VM
```

继电器只控制低压 LED 负载。MQ2 使用 5V 时，AO 必须经分压后进入 GPIO2，最大不得超过 3.3V。

## 3. 安全启动与自动联动

首次完整固件包含五类执行器驱动，个别武装开关均打开，但不允许上电瞬间动作：

1. 初始化时先锁定风扇 0%、继电器断开、蜂鸣器低电平、RGB 熄灭和舵机安全位置。
2. 进入 `5000ms` 启动保护窗；此期间仅采样、判断和上报。
3. 保护窗结束后，一次性进入整屋自动联动。
4. 安全传感器无效或过期时执行故障安全状态，不继续保留普通自动动作。
5. 安全覆盖永远高于手动命令；手动停止蜂鸣不等于关闭安全报警。

`hardwareVerified` 只表示实物证据，不因为软件已武装而自动改为 `true`。GPIO46 RGB 保留 `rgbHardwareVerified=false`，页面明确显示“命令已应用/实物尚未验证”。

## 4. 固件执行器基线

- 风扇：GPIO11 LEDC PWM，0–100% 限幅，启动默认 0%。
- 舵机：GPIO9，50Hz，使用可读的情境位置到角度映射，启动位置为安全关闭。
- 继电器：GPIO12，低压 LED 模型负载，在设为输出前先写入“断开”逻辑。触发极性保留为显式配置常量。
- 蜂鸣器：GPIO13 高电平有效，支持安全持续/间歇与 800ms 手动短测。
- RGB：GPIO46，12 颗、GRB/800kHz、低亮度状态色；保留实物未验证标记。

## 5. 三条实时链路

### 主路：Web Serial

```text
N16R8/CH340 -> Chrome/Edge Web Serial -> Dashboard
                                     -> 后续 WSS/MQTT -> 其他浏览器
```

页面必须由用户点击“连接开发板”触发授权，串口速率 115200。页面只把本项目的新鲜 `hello/telemetry` 当作真板在线证据。

### 备用：Python 网关

`tools/n16r8_gateway.py` 同时支持：

- `--mock-board`：显式模拟。
- `--serial-port /dev/cu.usbserial-XXX`：真板串口。
- 未给端口时只在明确真板模式中尝试自动发现 CH340。

### 后续：WSS/MQTT

公网不复用相邻项目的 `project/profileId/topic`。浏览器令牌和 MQTT/STT 凭据不进入 Git 或前端 JavaScript。

## 6. Dashboard 信息架构

首屏是操作台，不是宣传落地页：

1. 顶部状态：开发板、Web Serial、WebSocket、MQTT、数据新鲜度、当前情境。
2. 房屋热区：按家庭区域显示传感器、执行器和当前安全事件。
3. 可解释侦探：覆盖率、匹配度、支持/反对/缺失证据。
4. 传感器自检：数值、有效性、数据年龄、GPIO 和情境参与状态。
5. 安全原因：使用 `telemetry.alerts` 作为真值，说明传感器、GPIO、阈值和真实动作。
6. 六种情境：`detect/study/rest/ventilation/energy/custom`，命令必须带 ID 并等待同 ID `ack`。
7. 执行器调试：风扇、舵机、蜂鸣器、RGB、继电器的白名单命令，安全覆盖期间不得生效。
8. 设备注册、语音文本测试、当前状态和最近事件日志。

## 7. 验收边界

- Mock 测试证明 UI/协议，不代表真实硬件。
- 网页显示命令已发送、网关已转发、板端已 `ack`和实物可见动作为四个不同层次。
- 本阶段可完成软件、测试和 PIO 纯编译；带载烧录前必须关闭外接电池并释放串口。
- 烧录仅写 PIO 应用区 `0x10000`，独立 `verify_flash`，不擦除或改写 NVS、Wi-Fi、小智激活、bootloader 或分区表。
